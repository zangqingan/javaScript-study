1、js数据类型定义及分类

JavaScript 语言的每一个值,都属于某一种特定的数据类型(也可以叫语言类型)。一般使用一个变量存储,所以一般也就是检查变量的数据类型。
JavaScript共有八种基本数据类型,分别是 Undefined、Null、Boolean、Number、String、Symbol、BigInt、Object。 其中 Symbol 和 BigInt 是ES6 中新增的两种数据类型。虽然JavaScript只有8中数据类型,但是它是松散类型的所以不用担心不能表示全部数据。
这八种又可以分为两类:
    es5中前五种:Undefined、Null、Boolean、Number、String,以及es6的symbol、BigInt共七种都是基本数据类型(也叫原始类型)。
    而对象Object是一种复杂数据类型(也叫引用数据类型),它是一种无序的名值对集合。
所以js中是定义了7种基本数据类型,1种引用数据类型。
    Number 数值类型-用于任何类型的数字:整数或浮点数,在 ±(2的53次方-1) 范围内的整数。
    String 字符串类型-也就是我们常说的文本 "Hello World" , 'hi' 必须使用双/单引号包裹起来。
    Boolean 布尔类型-表示真伪的两个特殊值,即true(真)和false(假)。
    Null 表示空值,即此处的值为空。
    Undefined 表示“未定义”或不存在,即此处暂时没有任何值。
    Symbol 符号类型-代表创建后独一无二且不可变的数据类型,它主要是为了解决可能出现的全局变量冲突的问题。
    BigInt 用于任意长度的整数。是一种数字类型的数据,它可以表示任意精度格式的整数,使用 BigInt 可以安全地存储和操作大整数,即使这个数已经超出了 Number 能够表示的安全整数范围。
    Object 对象类型-各种值组成的集合。

4.Base64 编码

所谓 Base64 就是一种编码方法,可以将任意值转成 0-9、A-Z、a-z、+和/这64个字符组成的可打印字符。 使用它的主要目的,不是为了加密,而是为了不出现特殊字符,简化程序的处理。 JavaScript 原生提供两个 Base64 相关的方法,不过要注意,这两个方法不适合非 ASCII 码的字符,会报错 btoa() 任意值转为 Base64 编码 atob() Base64 编码转为原来的值 要将非 ASCII 码字符转为 Base64 编码,必须中间再插入一个转码环节。 btoa(encodeURIComponent(str)) decodeURIComponent(atob(str))

2. Number 数值

1.数值类型定义

js中所有数值都是Number类型,js不区分整数和浮点数,都是以64位的IEEE 754浮点数形式储存值。 所以,1与1.0是相同的,是同一个数。 根据国际标准 IEEE 754,JavaScript 浮点数的64个二进制位,从最左边开始,是这样组成的。 符号位+指数部分+小数部分 第1位 符号位,0表示正数,1表示负数 第2位到第12位(共11位) 指数部分 第13位到第64位(共52位) 小数部分(即有效数字) 符号位决定了一个数的正负,指数部分决定了数值的大小,小数部分决定了数值的精度。 JavaScript数值范围为2的1024次方到2的负1023次方(开区间),超出这个范围的数无法表示。 如果一个数大于等于2的1024次方,那么就会发生“正向溢出”,即 JavaScript 无法表示这么大的数,这时就会返回Infinity。 如果一个数小于等于2的-1075次方(指数部分最小值-1023,再加上小数部分的52位),那么就会发生为“负向溢出”,即 JavaScript 无法表示这么小的数,这时会直接返回0。 JavaScript 提供Number对象的MAX_VALUE和MIN_VALUE属性,返回可以表示的具体的最大值和最小值。 Number.MAX_VALUE // 1.7976931348623157e+308 Number.MIN_VALUE // 5e-324 JavaScript数值进制的表示方法 二进制 有前缀0b或0B的数值,0b11 // 3 八进制 有前缀0o或0O的数值,或者有前导0、且只用到0-7的八个阿拉伯数字的数值。0o377 // 255 十进制 没有前导0的数值如 10 十六进制 有前缀0x或0X的数值。0xFF // 255 一些特殊的数值 NaN是 JavaScript 的特殊值,表示“非数字”(Not a Number)用来表示本来是要返回数值的操作但是失败了而不是抛出错误。 注意NaN不等于任何值,包括它本身。NaN === NaN // false。可以使用isNaN(Number)函数检测一个值是不是NaN。

2.常用方法

Number.parseInt(str,"进制")方法用于将字符串转为整数,如果字符串的第一个字符不能转化为数字则返回NaN。 所以,parseInt的返回值只有两种可能,要么是一个十进制整数,要么是NaN。 Number.parseFloat()方法用于将一个字符串转为浮点数。 Number.isNaN()方法可以用来判断一个值是否为NaN,注意只对数值有效,如果传入其他值,会被先转成数值。如果传入的参数不能转换为数值则返回true。isNaN(NaN) // true isNaN(123) // false

3. BigInt 大整数

在JavaScript中 “number” 类型无法表示大于 (2的53次方-1)(即 9007199254740991),或小于 -(2的53次方-1) 的整数。这是其内部表示形式导致的技术限制。 所以在 ES2020 新增了BigInt就是用来表示任意长度的整数,通过将 n 附加到整数字段的末尾来创建 BigInt 值。用的比较少而且有兼容问题。
// 尾部的 "n" 表示这是一个 BigInt 类型 1234 // 普通整数 1234n // BigInt const bigInt = 1234567890123456789012345678901234567890n; 注意:BigInt 与普通整数是两种值,它们之间并不相等。 typeof运算符对于 BigInt 类型的数据返回bigint

4. Boolean 布尔值

代表“真”和“假”两个状态。有两个字面量值:“真”用关键字true表示,“假”用关键字false表示。 布尔值只有这两个值主要用来做程序流程的控制逻辑判断而且是区分大小写的。 例子 var bool = true;
布尔值往往用于运算符比较返回和程序流程的控制判断。

5. Null 空值

只有一个值即特殊值null,专门用来代表“无”、“空”或“值未知”的特殊值。 逻辑上讲它的值是一个空对象指针,即它指向一个空对象{},所以使用typeof操作符返回"object"。 用处:要定义一个将来保存引用类型的变量时,可以先使用null初始化。也就是说定义一个变量接收一个对象,而对象没有时可以先用null赋值给变量。

6. Undefined 未定义

含义是未被赋值,只有一个值undefined ,使用var或let声明一个变量但是没初始化赋值时返回的就是undefined,读取对象没有赋值的属性时也是undefined,函数没有返回值时,默认也返回 undefined。 未定义(未被赋值)它是一种状态,转为数值时为 NaN。
null与undefined都可以表示“没有”,含义非常相似。区别在于null是一个表示“空”的对象,转为数值时为0 =>> Number(null) // 0,
而undefined是一个表示"此处无定义"的原始值,转为数值时为NaN =>> Number(undefined) // NaN。

7. Symbol 符号

ECMAScript新增的数据类型,表示独一无二的值。作用是确保对象属性名都是独一无二的,避免属性名重名冲突。(这是因为在ES5 的对象属性名都是字符串，这容易造成属性名的冲突。)
1.基本用法

符号需要使用Symbol()函数初始化:let sym = Symbol();,可以传入一个字符串参数作为对符号的描述,之后可以通过这个描述来调试代码。
注意:它不能使用new关键字,这是因为生成的 Symbol 是一个原始类型的值,不是对象。 Symbol 值不能与其他类型的值进行运算，会报错。 Symbol 值作为对象属性名时,不能用点运算符要用方括号。 let s1 = Symbol('foo'); let s2 = Symbol('bar'); s1 // Symbol(foo) s2 // Symbol(bar) s1.toString() // "Symbol(foo)" s2.toString() // "Symbol(bar)"

2.使用场景

1.作为属性: let sy = Symbol("key1"); // 写法1 let syObject = {}; syObject[sy] = "kk"; console.log(syObject); // {Symbol(key1): "kk"} 写法2 let syObject = { [sy]: "kk" }; console.log(syObject); // {Symbol(key1): "kk"} // 写法3 let syObject = {}; Object.defineProperty(syObject, sy, {value: "kk"}); console.log(syObject); // {Symbol(key1): "kk"} 2.定义常量: const COLOR_RED = Symbol("red"); const COLOR_YELLOW = Symbol("yellow"); const COLOR_BLUE = Symbol("blue"); function getConstantName(color) { switch (color) { case COLOR_RED : return "COLOR_RED"; case COLOR_YELLOW : return "COLOR_YELLOW "; case COLOR_BLUE: return "COLOR_BLUE"; default: throw new Exception("'Can't find this color'"); } } getConstantName(COLOR_RED)

8. Object 对象

1.对象概述
js是一种基于对象的语言,js对象是一个由很多属性和与之对应的属性值组成的集合,这些属性(property)也称为键,它们之间使用用逗号分隔,对应的值可以是其它任意类型的。 如果一个属性的值为函数,通常把这个属性称为“方法”,它可以像函数那样调用。 简单说对象就是一组“键值对”key-value的集合、一种无序的复合数据集合。
对象的所有键名都是字符串或者ES6新增的符号类型(数值时是自动转成字符串的),所以属性名的引号可加可不加。
2.对象属性的操作

读取对象的属性:有两种方法,一种是使用点运算符,还有一种是使用方括号运算符。 请注意,如果使用方括号运算符,键名必须放在引号里面字符串形式,否则会被当作变量处理。 var obj = { p: 'Hello World' }; obj.p // "Hello World" obj['p'] // "Hello World" 赋值或修改: var obj = {}; obj.foo = 'Hello'; obj['bar'] = 'World'; 删除: delete命令用于删除对象的属性,删除成功后返回true。 var obj = { p: 1 }; Object.keys(obj) // ["p"] delete obj.p // true obj.p // undefined Object.keys(obj) // []

2、js值数据类型的判断方法

因为JavaScript 是一种送散的动态类型语言,变量是没有类型限制,可以随时赋予其它任意类型值。所以需要一种方法来确定任意变量的数据类型是什么？
在JavaScript 中常见有4种方法,可以确定一个值到底是什么类型。
    1. typeof运算符
    2. instanceof运算符
    3. Object.prototype.toString方法
    4. constrcutor对象

1. typeof运算符

typeof运算符可以返回一个值的数据类型,返回值是字符串类型的即==》typeof typeof 值;得到的是 "string"。
使用它可以判断一个变量是否是原始类型,确切地说是判断一个变量是否是6个基础数据类型。至于null、对象不应该使用它。
    typeof 123 // "number"
    typeof 123n // "bigint"
    typeof '123' // "string"
    typeof sym // "symbol"
    typeof false // "boolean"
    typeof undefined // "undefined"
    typeof null // "object"
    typeof fun // "function"
    typeof {} // "object"
    typeof [] // "object"

2. instanceof运算符

instanceof运算符只能用来判断一个变量是什么引用类型的(对象、数组或者其它引用类型),它返回一个布尔值表示对象是否为某个构造函数的实例。 其内部运行机制是判断在其原型链中能否找到该类型的原型。
    实例 instanceof 对象
    var v = new Vehicle();
    v instanceof Vehicle // true
应用:利用instanceof运算符、还可以巧妙地解决、调用构造函数时、忘了加new命令的问题。

3. constrcutor对象

构造器对象它指向的是对象的构造函数本身、所以通过判断变量的构造器对象属性是否等于对应的钩子函数即可判断它的类型。
     2.constructor === Number; // true
     true.constructor === Boolean; // true
     'str'.constructor === String; // true
     [].constructor === Array; // true
     function() {}.constructor === Function; // true
     {}.constructor === Object; // true

4. Object.prototype.toString方法

它会返回当前对象对应的字符串形式,通过对象冒充call()方法让任意值调用这个方法、同时防止其它继承Object原型的toString重写了,因此可以用来判断一个值的数据类型。
不同数据类型的Object.prototype.toString方法返回值如下。
    数值:返回[object Number]。
    字符串:返回[object String]。
    布尔值:返回[object Boolean]。
    undefined:返回[object Undefined]。
    null:返回[object Null]。
    数组:返回[object Array]。
    arguments 对象:返回[object Arguments]。
    函数:返回[object Function]。
    Error 对象:返回[object Error]。
    Date 对象:返回[object Date]。
    RegExp 对象:返回[object RegExp]。
    其他对象:返回[object Object]。
    中括号内第一个字符都是 object、 第二个字符表示该值对应的构造函数。
    console.log(Object.prototype.toString.call(123)); //[object Number]
    console.log(Object.prototype.toString.call('123')); //[object String]
    console.log(Object.prototype.toString.call(undefined)); //[object Undefined]
    console.log(Object.prototype.toString.call(true)); //[object Boolean]
    console.log(Object.prototype.toString.call({})); //[object Object]
    console.log(Object.prototype.toString.call([])); //[object Array]
    console.log(Object.prototype.toString.call(function(){})); //[object Function]
    console.log(Object.prototype.toString.call(null)); //[[object Null]]

3、js数据类型转换

JavaScript 是一种动态类型语言，变量没有类型限制，可以随时赋予任意值。将一个变量的数据类型转换为其它的数据类型,主要是String,Number,Boolean这三种之间的转换。 类型转换: 又有强制类型转换和隐式类型转换两种:
强制转换主要指使用Number()、String()和Boolean()三个函数,手动将各种类型的值,分别转换成数字、字符串或者布尔值。

1.其它数据类型转为--->String

方法一 调用被转换数据类型的tostring()方法(也是一个函数几乎所有值都有),它会返回当前值的字符串等价物。调用谁的什么方法就是 谁.方法名() 注意 该方法不会影响原变量,而是将转换后的结果返回,null和undefined这两个值没有tostring()方法,调用会报错。
    方法二 如果不确定一个值是不是null或undefined,就使用String()转型函数,调用什么什么函数就直接写 函数名()；转谁就把谁当作参数写在函数的小括号()里。 注意 该方法也不会影响原变量,而是将转换后的结果返回。 原则是如果值有tostring()方法就调用这个方法并返回结果。 对于null和undefined则是直接转换为字符串的"null","undefined"。
转换规则：
    数值：转为相应的字符串。
    字符串：转换后还是原来的值。
    布尔值：true转为字符串"true"，false转为字符串"false"。
    undefined：转为字符串"undefined"。
    null：转为字符串"null"
    普通对象返回类型字符串 "[object Object]"
    数组返回逗号分隔的字符串形式

2.其它数据类型转换为--->Number

方法一 调用Number()函数,用法和String()函数类似,可以将任意类型转换为数值类型。
    字符串--->数值 1.纯数字字符串直接转为对应的数字。2.有非数字转为NaN。3.空串或者全是空格转为0
    布尔---->数值 true为1,false为0.
    NULL---->数值 null-0
    undefined--->数值 undefined->NaN。
    对象--->数值 {}->NaN
方法二 专门针对字符串转换为数值的方法。
    parseInt(),parsefloat()函数,将一个字符串转换为一个整数/浮点数。
    从左往右找,开头就是字母返回NaN,不是字母则一次往右找直到第一个非数字停止。
    对于非string类型先转换为string类型再操作。

3.其它数据类型转换为----->Boolean

使用Boolean()转型函数,用法和String()函数类似,可以将任意类型转换为布尔值。 数字-------->布尔 除了NaN,±0为false,其它都为true。 字符串------>布尔 除了空串为false,其它都为true。 null-------->布尔 null为false。 undefined--->布尔 undefined为false。 对象-------->布尔 都为true。 注意,空数组([])和空对象({})对应的布尔值也是true。

隐式类型转换遇到以下三种情况时,JavaScript 会自动转换数据类型,即转换是自动完成的,用户不可见。
第一种情况,不同类型的数据互相运算加减乘除啥的。123 + 'abc' // "123abc" 第二种情况,对非布尔值类型的数据求布尔值。 第三种情况,对非数值类型的值使用一元运算符（即+和-）。 自动转换的规则是这样的预期什么类型的值,就调用该类型的转换函数。本质上还是使用强制类型转换的转换函数不过是JavaScript引擎帮忙后台处理了。
